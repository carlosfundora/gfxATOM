## 2024-05-13 - [Optimize Hot Path Redundant Function Call in topK]
**Learning:** Found an expensive string-matching loop function (`is_rocm_aiter_fusion_shared_expert_enabled`) called repeatedly on per-token hot paths (`rocm_aiter_topk_softmax_impl`, etc.). The check was mathematically redundant because it already guarded `num_fused_shared_experts` initialization, meaning checking `num_fused_shared_experts > 0` at runtime inherently proves fusion is enabled. Additionally, caching global config lookups in `__init__` is dangerous as they can change dynamically at runtime via context managers.
**Action:** Avoid placing complex string-matching iterations or global config lookups in inner loops. Utilize pre-computed state variables safely set during initialization where possible, and avoid caching dynamic config properties directly.
## 2024-05-14 - [Optimize Hot Path Redundant Function Call in Models]
**Learning:** Found an expensive string-matching loop function (`is_rocm_aiter_fusion_shared_expert_enabled`) called repeatedly on per-token hot paths in model forward passes (e.g. Qwen3Next, Glm4Moe, DeepseekV2). Since `self.shared_experts` (or `self.shared_expert`) is explicitly initialized in `__init__` if and only if fusion is disabled, the expensive runtime check can be avoided by checking `getattr(self, 'shared_experts', None) is not None`.
**Action:** Remove the redundant function calls in the forward passes and replace them with a simple attribute check. Added comments explaining the optimization so the logic is clear to future developers.
## 2024-05-15 - [Optimize Numpy Hot Loops in Audio Fallback]
**Learning:** In audio inference hot loops, avoiding allocations is paramount. When performing in-place updates with `.astype` casting (like `np.where(...).astype(...)`), `np.multiply(out=...)` mutates the array without duplicating it. Furthermore, autoregressive loop positions such as `position_ids` can be pre-allocated once over the whole context sequence length plus max generation size (`seq_len + max_tokens`), then sliced in the loop (`pos_ids_full[:, current_seq_len - 1:current_seq_len]`), avoiding the per-token `np.arange` or `np.array` allocation overhead.
**Action:** Replaced repetitive per-token masking/allocations in `_np_rep_penalty` with `np.multiply(..., out=scores)`. Pre-allocated `pos_ids_full` in `_generate_onnx_cpu` and replaced the inner loop dynamic allocations with zero-copy slicing.
