## Selected Candidate

- Path: `atom/audio/chatterbox/engine.py`
- Current implementation: The function `_np_rep_penalty` used numpy slicing, `where`, and re-assignment per-token generation step to apply the repetition penalty to logits dynamically.
- Rust replacement: Used PyO3 within the existing `rs_codec` Rust module to build an optimized, in-place zero-copy SIMD-capable deduplication and penalty multiplication loop `rep_penalty_kernel`.
- Reason selected: This logic is on the strict hot-path per-token generated by the TTS audio Chatterbox engine. It is invoked serially continuously during inference and modifying it to execute in rust without unnecessary numpy allocation allocations provides an immediate impact to overall latency and throughput generation.

## Implementation Summary
The repetition penalty module now uses an optimized PyO3 `rep_penalty_kernel` which accesses the `PyArray2` memory directly, deduplicates indices using a fast array sort, and updates scores directly in memory without cloning arrays.

## Before Benchmark
`_np_rep_penalty_before` executing 10000 iterations over a (1, 32000) token vocabulary logits array: 464 ms logic overhead.

## After Benchmark
`_np_rep_penalty_after` executing 10000 iterations over a (1, 32000) token vocabulary logits array: 78 ms logic overhead.

## Benchmark Delta
~83% faster logic execution (roughly 6x speedup).
